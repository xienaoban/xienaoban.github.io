Doxygen 安装使用

安装

也可以从源码编译安装, 源项目是 CMake 项目. 不过源码安装会导致使用 apt 安装的 GUI 找不到 CLI 安装位置, 还是 apt 方便. 清华的 apt 源貌似没有 doxygen, 后来改用阿里云的源就好了.

使用

GUI 程序使用的典型流程

以分析 Doxygen 项目本身为例, 目标是获取其函数调用关系. 先打开 GUI 程序:

  1. 确定工作目录, 填写项目名称 (貌似不填也无所谓), 选择源码目录 (CMake 项目的根目录即可), 记得勾选 Scan recursively, 即子目录的源文件也会被检索. 然后确定一下文档的输出目录.

  2. 提取模式勾选 All Entities, 具体比第一个多哪些内容我还没看, 对运行时间的影响和提取的内容待考证.

  3. 选择输出格式, 我的目标不是输出文档而是提取函数调用关系, 仅导出 XML 就够了, 导出 HTML 和 LaTex 等速度很慢, XML会快上不少 (但也还是很慢).

  4. 图表选 dot tool, 关键是把最后俩选项勾选以获得函数调用信息 (如果下面的框框都是灰色的, 就重新选择一下 dot tool).

  5. EXTRACT 全部勾选, 这些内容和函数调用应该都有关系 (根据左下角的描述, 其中一个特别耗时, 我忘了是哪个).

  6. dot 的设置如图, dot 是用来生成调用图的. CALL_GRAPH 和 CALLER_GRAPH 分别是函数的调用视图和被调用视图. DOT_PATH 是 dot 安装地址 (dot 就是 apt 安装的 graphviz). dot 貌似仅对 HTML 等起作用? 以后我再看看仅导出 XML 是不是可以不用 dot.

最后转到 Run 选项卡运行即可.

Doxyfile 与 CLI 使用

我想把 Doxygen 作为 CMake 项目源码的静态分析工具. 由于需要用脚本自动化执行, 用 GUI 肯定不现实, 于是又研究了下这个 CLI 怎么用.

除了源/头文件之间的依赖关系, 我还需要每个源/头文件之间的函数、全局变量、类等等所有的调用关系, 例如 文件 A 中的 函数 foo 调用了 文件 B 中的 函数 bar. 同时我不想让分析过程耗时太久, 所以尽量把不相关的选项全部关了. 针对上述需求，整理了份 Doxyfile 模板:

使用时仅需要把上述代码段中第 90 行的 <input> 替换为你指定的所有源文件或源文件目录即可。例如改为:

  • 注: Doxygen 似乎不关心你的项目是 CMake 组织的、Makefile 组织的还是什么框架都没有用, 我们只需要把所有源文件或所有源文件所在目录告诉它, 它会根据源/头文件名自动去匹配依赖 (所以如果项目在不同目录有同名文件, Doxygen 可能识别出错).

然后在该目录执行指令 doxygen Doxyfile 即可 (Doxyfile 文件名并不强制, 如果你的 Doxyfile 文件名不叫 Doxyfile, 那把指令改成 doxygen 你的文件名 即可).

其输出是一堆 XML 文件. 其中 index.xml 给每个文件的每个函数、每个类、变量等等提供了一个唯一 id. 其余每个文件代表一个源文件、头文件或一个目录的信息. 里面有每个文件之间的类、函数、全局变量等等的调用关系 (其中涉及函数、类、变量调用关系的标签是<references> 和 <referencedby>, 分别是 "调用" 与 "被调用"). 整个文件格式还是蛮清晰易懂的.
